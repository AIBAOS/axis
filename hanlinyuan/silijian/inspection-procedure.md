## 📋 巡检检查清单（必须逐项勾选）

### 第一步：阅读上下文（必须！）
- [ ] 读取最近 50 条 Discord 聊天记录
- [ ] 识别已完成的任務（关键词："已完成"、"✅"、"commit"）
- [ ] 识别待办任务（关键词："待实现"、"⏳"、"待开发"）
- [ ] 识别阻塞项（关键词："阻塞"、"等待"、"内阁裁决"）

### 第二步：核对 GitHub（必须！）
- [ ] `git fetch origin` 获取最新提交
- [ ] `git log --oneline -10` 查看最近 commit
- [ ] 记录最新 commit hash、时间、提交人
- [ ] 对比 README 中的进度表与实际代码

### 第三步：更新记忆（必须！）
- [ ] 更新 `memory/YYYY-MM-DD.md` 记录今日进度
- [ ] 如有重大变更，同步更新 `MEMORY.md`
- [ ] 记录 commit hash 和完成时间

### 第四步：派发任务（仅当确认待办）
- [ ] 确认任务未被完成（查聊天记录 + GitHub）
- [ ] 使用正确格式 @对应部门（第一行单独成行）
- [ ] 写明【角色】【任务】【背景】【要求】【格式】
