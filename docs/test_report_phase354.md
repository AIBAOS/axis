## 发现的问题

### Bug #48: 后端 storage_pool_create.rs 缺少最小长度检查

**严重度:** 🟡 低

**问题:**
- 后端只检查 `name.is_empty()` 但没有检查最小长度
- 前端允许 1 字符名称
- 但后端实际也拒绝了空名称，所以影响较小

**评估:** 空名称已被拒绝，无需修复

### Bug #49: 后端 storage_volume_create.rs 缺少最小长度检查

**严重度:** 🟡 低

**问题:**
- 后端只检查 `name.is_empty()` 但没有检查最小长度
- 前端允许 1 字符名称
- 但后端实际也拒绝了空名称，所以影响较小

**评估:** 空名称已被拒绝，无需修复

### Bug #50: 后端 files_upload.rs 缺少文件名长度限制

**严重度:** 🟠 中等

**问题:**
- 前端限制文件名最大 255 字符
- 后端没有长度检查
- 超长文件名可能导致文件系统问题

**修复建议:** 后端添加文件名长度检查 (255 字符)

### Bug #51: 前端 validateFilename 未显式检查 null 字节

**严重度:** 🟡 低

**问题:**
- 后端检查 `\0` (null 字节)
- 前端只检查控制字符 (0x00-0x1f)，已包含 null 字节

**评估:** 控制字符检查已包含 null 字节，无需修复
